Using the above scenarios, we see the distinction between `state` and `observation`.
State is the true representation of the environment at a given time. Observation is what we get to see. And the observation can be partial.
Hence, even if the future can be fully determined based on the current state and action, it might still require us to have access to previous observations to be able to figure out the future.
In this case, the past observations are essentially being used to figure out the missing data in the current state.

## Reward and Value Function

An agent which interacts with an environment is given a numeric reward on each time step which depends on the state the agent is in,
and the action that is taken by the agent. The reward provided is a property of the environment. Rewards can also be stochastic i.e.
for the same chosen action in a given state, the agent might get different rewards at different times. The objective of the agent is to collect
as much reward as possible over time. The rewards guide the agent in learning what actions to take in different states.

Once you have answered the above set of questions, I hope it would have become clear that taking an action which yields the highest 
immediate reward (referred to as a greedy policy), is generally not optimal. Instead, what one should care about is long term rewards.
For example, in a game of chess, it might be possible to gain a large immediate reward by taking a piece (assuming that positive rewards
are provided whenever the agent takes a piece), but strategically it might be the wrong move causing the agent to eventually lose the
game and get a very large negative reward.

We should also notice that rewards can be stochastic. Hence, we cannot be sure of the reward we get if we take an action. Therefore, most
of the time we would be talking about <ChatGptLink prompt="Explain expectation in probability theory">expected</ChatGptLink> reward.

How good a state is is measured by its value function. The value function is the expected total reward from that state given a particular policy (policy is the strategy of a player).
We can obviously have a value function for a state which does not depend on a particular policy. In that case, we could calculate the 
expected value of that state by calculating the expectation over all the possible policies. The distribution of policies might be
considered uniform. But in actual learning process, we care about how good is a state for a particular policy, not in general. This will
become evident when we look into algorithms for policy learning.

## Model

The model of the environment mimics how the environment would behave when an action is taken in a given state. Hence, the learning
algorithm can query the model for the next state and reward as a result of taking an action without actually taking that action while
interacting with the actual environment. It seems quite obvious that having a model can be very beneficial in the learning process
since we can get to know what the environment will do. For example, if the agent was trying to learn how to drive a car, a model of
the world could tell it what would happen if a particular steering action was taken. However, it also seems quite unlikely that such
a model would be available for most of the problems involving complex environments.
